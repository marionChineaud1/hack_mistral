# 6. Demo video storyboard

**Target length:** 90 seconds

## Format

Record a **hybrid product demo**, not a live presentation:

1. Capture the Mistral Vibe investigation once from URL submission through the final link.
2. Capture the Vera report separately, using the prepared air-conditioning report.
3. Edit the clips together with a short voice-over and a few on-screen captions.

This keeps the video credible because it shows the real product, while avoiding dead time, unpredictable live retrieval, and accidental exposure of private browser tabs or credentials.

## Before recording

- Use a clean browser profile and close unrelated tabs, notifications, terminals, and developer tools.
- Record at 16:9, 1080p. Keep browser zoom consistent, ideally 100%.
- Use the same X post and resulting Vera report throughout.
- Ensure the report page opens and source links are visible before recording.
- In Mistral Vibe, make the tool calls and short statuses visible. Do not show secrets, configuration, or raw logs.
- Record a little extra before and after each action so cuts are clean.

## Recording the report

Use the existing report in this order:

1. **Claim under review** and the **Quick conclusion**.
2. **Browse the sources**, showing primary and secondary source labels.
3. **What makes this reading stronger or weaker**, including a limitation.
4. **What supports the claim** beside **What qualifies or challenges it**.
5. **How the wording shapes the message** to demonstrate separate rhetorical analysis.
6. **What could change this reading** for the closing shot.

Avoid scrolling through the full report. Use short, intentional cuts between these sections.

## Editing notes

- Keep the pace brisk. Cut any loading period longer than one second.
- Highlight the cursor only for the URL submission, report link, and a few key sections.
- Add captions because hackathon videos are often watched without sound.
- Use a subtle zoom or crop for small UI details, rather than recording at a lower resolution.
- Do not claim that Vera proves a claim true or false. Use “supports,” “qualifies,” “contradicts,” “uncertain,” and “what could change the conclusion.”
- Use the real report and real statuses. Do not substitute mock evidence.

## Optional 30-second cut

For a short submission teaser, retain only:

1. 0:00–0:06, the problem and URL submission;
2. 0:06–0:14, the visible Vibe statuses;
3. 0:14–0:22, the architecture connection;
4. 0:22–0:30, the report conclusion, sources, and closing line.

## Final voice-over script

“Viral claims often arrive before the context needed to understand them. Vera turns an X post into a transparent investigation. In Mistral Vibe, its specialised workflow traces the origin, checks live institutional evidence, recovers missing context, and analyses framing. MCP tools connect that workflow to live research, while the completed investigation is saved as structured, citable evidence. Vibe returns a concise conclusion and a link to the full report, where readers can inspect the sources, limitations, and what could change the conclusion. Vera does not decide the truth. It shows the work needed to reach a responsible conclusion.”
